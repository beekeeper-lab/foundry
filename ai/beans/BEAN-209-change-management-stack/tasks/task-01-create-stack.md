## Description

Create the `ai-team-library/stacks/change-management/` directory with comprehensive stack files covering:

1. **fundamentals.md** — ADKAR framework, change management principles, organizational readiness assessments
2. **stakeholder-management.md** — Stakeholder mapping templates, impact analysis, engagement strategies
3. **communication-plans.md** — Communication planning templates, channel strategies, messaging frameworks
4. **training-and-adoption.md** — Training curricula design, adoption KPI tracking, learning path design
5. **risk-and-rollback.md** — Rollback triggers, change failure management, risk assessment

Each file must follow the standardized template pattern observed in existing stacks:
- Defaults table with alternatives
- Do/Don't lists
- Common Pitfalls
- Checklist
- Practical examples/templates

## Inputs

- Existing stacks for template pattern (e.g., `ai-team-library/stacks/iso-9000/`)
- Bean acceptance criteria
