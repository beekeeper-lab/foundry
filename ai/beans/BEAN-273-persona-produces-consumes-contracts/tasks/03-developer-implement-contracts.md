## Goal

Implement the four code/content changes the Architect's ADR specifies:

1. **Registry file.** Create `ai-team-library/contracts/artifact-types.yml`
   with the full registry content from BA's deliverable. Use exactly the
   field names BA specified (`name`, `description`, `format`,
   `required-fields`, `template-path`).
2. **Persona contracts.** Per ADR-013, the contract goes in a **sibling
   `contracts.yml`** next to each `persona.md` (NOT frontmatter — see the
   ADR's parser-fragmentation rationale). Create `contracts.yml` next to
   each of the five core library persona files:
   - `ai-team-library/personas/ba/persona.md`
   - `ai-team-library/personas/architect/persona.md`
   - `ai-team-library/personas/developer/persona.md`
   - `ai-team-library/personas/tech-qa/persona.md`
   - `ai-team-library/personas/team-lead/persona.md`
   Use the `produces:` / `consumes:` lists from BA's writeup.
   *Out of scope:* extended personas and the kit's `.claude/shared/agents/*.md`.
   Those propagate via a separate kit/regen step (follow-up bean).
3. **Loader.** Per the ADR's choice, either extend
   `foundry_app/services/library_indexer.py` or add
   `foundry_app/services/contracts_loader.py`. The loader must:
   - Load `artifact-types.yml` and validate every entry has all required
     fields. Raise a clear error naming the offending entry on a missing
     field.
   - Parse persona frontmatter (use a small, well-tested helper — do not
     import a heavy markdown library; a regex split on the first two `---`
     lines is sufficient).
   - For each persona on a team, return a structured object exposing
     `produces`, `consumes`, and the resolved registry entries.
   - Validate that every type referenced by a persona exists in the
     registry; raise a clear error naming the persona + the missing type.
4. **Compiler emission.** Update `foundry_app/services/compiler.py` so the
   generated `ai/team/composition.yml` includes a top-level `contracts:`
   block in the shape the ADR fixed. The block lists per-persona
   `produces:` / `consumes:` and a flat `artifact-types:` reference list.

Add or extend Pydantic models in `foundry_app/core/models.py` if the loader
or compiler returns new structured data — keep models minimal and aligned
with existing patterns.

Wire the loader into the existing pipeline. The compiler stage must call
the loader before emitting `composition.yml`; if the loader raises, the
generation fails fast with a useful message.
