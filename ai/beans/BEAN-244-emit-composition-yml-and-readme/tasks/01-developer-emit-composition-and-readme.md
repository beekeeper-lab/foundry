## Goal

Make a freshly generated project self-describing and self-validating: emit `ai/team/composition.yml` (round-trippable source spec) and a starter `README.md` at the project root, so `validate-repo`'s structural check passes and BEAN-242's structural assertion goes green.

## Inputs

- `foundry_app/services/scaffold.py`
- `foundry_app/io/composition_io.py::save_composition` (existing helper)
- `foundry_app/__init__.py::__version__` (for README footer)
- BEAN-242 structural assertion — drives this work.

## Implementation

- Added `ai/team` to the `_AI_DIRS` list in `scaffold.py` so the skeleton carries the directory.
- After creating directories, `scaffold_project` now calls `save_composition(spec, root / "ai/team/composition.yml")`. The write is idempotent — only counted in `StageResult.wrote` when the on-disk content actually changed — so overlay/regen over an unchanged spec remains a no-op.
- A starter `README.md` is written at the project root with the project name, a description (from `spec.project.description` or a sensible default), a pointer to `CLAUDE.md`, a structure blurb, and a `Generated by Foundry vX.Y.Z on YYYY-MM-DD` footer. README emission is overlay-safe: if a README already exists, the user's version is preserved.
- `tests/test_scaffold.py` — bumped the wrote-count assertion from 12 → 15 (added 1 dir + 2 files) and added two new test classes: `TestCompositionSnapshot` (writes, roundtrip, idempotency, re-emits on spec change) and `TestStarterReadme` (writes starter, preserves user-customized README).
